Project 1 — Geo-Location Café Finder

A location-based café search tool built with vanilla JavaScript and the Google Maps API. You open it, it asks for your location, centres a map on you, and lets you search for cafés nearby in real time.

The technical challenge was handling the full async API cycle — permission request → geolocation → Maps JS API → Places API search → dynamic results rendering — without any framework doing the heavy lifting. Everything is manual.

Google Maps JS
interactive map that centres on your real location and updates as you search.
Places API
real-time nearby café data — name, rating, distance, open/closed status.
Geolocation API
handles browser permissions, fallback states, and dynamic map centering.
Async JS
manages the full API call cycle without blocking the UI — results render as they arrive.
Project 2 — This portfolio

The site you're on right now. Designed and built from scratch — dark, bold, no templates, no frameworks. Every layout decision, interaction, and line of CSS is hand-written.

It's gone through multiple redesigns. The current version is the one that actually feels like me — which took longer than any technical decision.

What building without frameworks actually teaches you

When React or Vue breaks, you Google it. When vanilla JS breaks, you have to understand what the browser is actually doing. These projects taught me how the web works — the event loop, async/await, DOM manipulation, API response handling — not just how to use a library that abstracts it.

That foundation is why picking up any framework later feels straightforward. You already know what it's doing underneath.
