What it is

A full-stack investment tracker where three separate layers — a PHP frontend, a SQLite database, and a Python analytics backend — talk to each other through the Yahoo Finance API. You enter a stock ticker, the system fetches live market data, calculates your ROI, and shows it on a dashboard.

The point wasn't the finance — it was building a real data pipeline end to end. Input → storage → processing → live output. That whole loop in one project.

The numbers and what they mean
15+ assets
tracked simultaneously with live prices pulled from Yahoo Finance API — stocks, crypto, whatever you add.
3-layer architecture
PHP handles the user interface and CRUD operations. SQLite stores everything with relational integrity. Python hits the API, runs the Pandas calculations, and writes results back. Each layer does one job.
Automated ROI
unrealised gains/losses and sector allocation calculated automatically every time the Python script runs — no manual spreadsheet work.

What I actually learned

That making three different languages and technologies talk to each other cleanly is harder than it looks. PHP and Python don't naturally know about each other — the database is the shared language between them. Getting that architecture right, where each layer has clear responsibility and doesn't bleed into another, was the main thing.

Also: Yahoo Finance API is free until it isn't. Rate limits are real.
